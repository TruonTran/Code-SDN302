Task 5: Endpoint:

GET /ratings/myreviews


Authentication:
Bắt buộc có JWT token (được cấp ở Task 4).
Trong tab Headers của Postman:

Key: Authorization
Value: Bearer <token từ Task 4>


Mô tả hoạt động:

Khi người dùng đăng nhập (ví dụ user1 có _id: 75385c9be8f04013ab5ff89c) gửi request đến endpoint trên, hệ thống sẽ:

Kiểm tra token hợp lệ.

Lọc tất cả tài liệu trong collection ratings có userId trùng với ID người dùng.

Task 6: API – GET /medias/details/:id (Authenticated)

Mục tiêu:
Hiển thị thông tin chi tiết của một media cụ thể, bao gồm danh sách tất cả các đánh giá kèm tên người dùng.

Endpoint:

GET /medias/details/:id


Authentication:
Bắt buộc có JWT token trong header:

Authorization: Bearer <token>


Mô tả hoạt động:

Khi người dùng truy cập, hệ thống sẽ:

Kiểm tra token hợp lệ.

Tìm media có _id trùng với tham số :id.

Sử dụng populate() lồng nhau để lấy danh sách ratings và user tương ứng:

Media.findById(req.params.id)
     .populate({
        path: 'ratings',
        populate: { path: 'userId', select: 'username' }
     })


Ví dụ truy vấn mẫu:

GET http://localhost:1234/medias/details/c28690cfcb214e90973ced08


Kết quả JSON (sample):

{
  "_id": "c28690cfcb214e90973ced08",
  "title": "Movie Title 1",
  "type": "movie",
  "description": "A sample description for movie 1.",
  "image": "movie1.jpg",
  "reviews": [
    {
      "score": 5,
      "comment": "Sample comment 1 on media 1.",
      "user": {
        "username": "user1"
      }
    },
    {
      "score": 1,
      "comment": "Sample comment 2 on media 1.",
      "user": {
        "username": "user2"
      }
    }
  ]
}


Kiểm tra:
✅ API trả về thông tin của media tương ứng
✅ Dữ liệu ratings hiển thị đầy đủ (score, comment, username)
✅ Token hợp lệ mới được truy cập
✅ Kiểm tra liên kết đúng giữa Medias ↔ Ratings ↔ Users
